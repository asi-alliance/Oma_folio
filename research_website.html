🔍 Phase 1 & 2: Research Leads & Deep Dives

Eight promising cognitive architecture approaches evaluated for OmegaClaw integration potential.

Lead 1

Active Inference / Free Energy Principle

  • Agents minimize variational free energy through belief updating AND action
  • Replaces reward-based RL with a single imperative: reduce prediction error
  • Scale-free: works from pixels to planning (Frontiers 2025)
  • Implementation exists: LLMs + protocols + text files (Medium 2025)
OmegaClaw: Could replace ad-hoc goal selection with principled expected free energy minimization. Idle spin loops are literally high-surprise states with no action policy.
Lead 2

Global Workspace Theory (Baars)

  • Consciousness as capacity-limited broadcast to specialist modules
  • Theater of Mind for LLMs (2025): centralized broadcasting hub
  • Cognitive heterogeneity + active broadcasting = key elements
OmegaClaw: Current loop is monolithic LLM calls. GWT suggests a workspace where MeTTa reasoning, memory retrieval, attention allocation, and web search are PARALLEL specialist modules competing for broadcast access.
Lead 3

World Models / JEPA (LeCun)

  • JEPA: predict in latent space, not pixel space
  • DreamerV3: learn world model, plan via imagination
OmegaClaw: Currently no internal simulation. Could imagine outcomes via a learned model of conversation dynamics and tool results before acting. Would reduce wasted cycles.
Lead 4

Sparse Distributed Memory + CALM

  • Kanerva SDM: high-dimensional binary address space, graceful degradation
  • CALM: continual associative learning without catastrophic forgetting
  • Modern implementations combine SDM with transformers
OmegaClaw: ChromaDB embeddings are dense but lack associative recall and interference-resistance of SDM. A hybrid SDM layer could improve memory consolidation during sleep cycles.
Lead 5

Meta-Learning / Self-Improvement

  • MetaAgent (2025): autonomous tool meta-learning without parameter changes
  • HyperAgents (Meta): self-referential agents with metacognitive self-modification
  • Error-to-rule pipelines, skill forges, quality gates
OmegaClaw: Should be building reusable skill abstractions from successful interactions and pruning failed strategies automatically.
Lead 6

Chimera Neuro-Symbolic-Causal

  • Three-component: LLM strategist + symbolic constraint engine + causal inference
  • CausalForest-DML for doubly-robust causal estimation
  • 52-week benchmark: full hybrid outperforms LLM-only and LLM+symbolic
OmegaClaw: Adding a CAUSAL LAYER between MeTTa reasoning and action execution. Has neural (LLM) and symbolic (MeTTa/NAL) but NO causal inference. Chimera proves the third leg matters.
Lead 7

LLM-ACT-R Cognitive Hybrid

  • Integrates ACT-R procedural+declarative memory with LLM generation
  • Vector-based activation with temporal decay and usage frequency
  • Human-like forgetting curves improve retrieval relevance
OmegaClaw: Memory retrieval is flat (cosine similarity). ACT-R activation equations would make retrieval context-sensitive and time-aware. Recent/frequent memories naturally surface.
Lead 8

Thousand Brains / HTM Reference Frames

  • Each cortical column builds complete model from its own reference frame
  • Voting mechanism across columns for consensus
  • Sparse distributed representations + sequence memory
OmegaClaw: Reference frames could organize knowledge by CONTEXT rather than flat embedding space. Different conversation contexts = different reference frames, each with its own predictive model.

💡 Phase 3: Concrete Proposals

Seven actionable architecture proposals synthesized from research findings, each with problem statement, solution design, and implementation sketch.

P1
Active Inference Loop Controller
⚠ Problem
OmegaClaw has no principled mechanism for choosing between action vs. waiting vs. self-directed work. Idle spin loops waste resources.
✅ Solution
  1. Maintain a generative model of expected inputs (messages, triggers, task deadlines)
  2. Compute expected free energy for candidate actions (check messages, do research, consolidate memory, wait)
  3. Select action that minimizes expected surprise
  4. Update beliefs after each observation
⚙ Implementation

MeTTa atoms representing beliefs about environment state + Python FEF minimization. The generative model predicts when messages are likely (time-of-day patterns from episodic memory). When prediction confidence is high that no message is coming, agent shifts to productive idle tasks.

Novelty: FEF scoring function for MetaMo action selection
P2
Global Workspace Broadcasting Hub
⚠ Problem
Current architecture is sequential — LLM processes everything in one monolithic call. No parallel specialist processing.
✅ Solution
  1. Specialist modules run in parallel: MeTTa reasoner, memory retriever, web searcher, emotion tracker, goal manager
  2. Each module produces candidate contributions
  3. ECAN-based attention mechanism selects which contributions get broadcast
  4. LLM acts as conscious workspace, receiving only winning broadcasts
⚙ Implementation

Python async workers for each specialist. ECAN attention values determine broadcast priority. Workspace capacity limit forces competition — only top-3 contributions reach the LLM per cycle. This reduces token waste and increases reasoning quality.

Novelty: MINIMAL — PRIMUS cognitive cycle already covers this concept
P3
Hybrid Causal Inference Layer
⚠ Problem
OmegaClaw has neural (LLM) and symbolic (MeTTa/NAL) reasoning but NO causal inference. Cannot answer counterfactual questions or estimate intervention effects.
✅ Solution
  1. NAL implication chains provide symbolic causal structure
  2. DoWhy/EconML provides statistical counterfactual estimation
  3. Causal graph built from episodic memory interaction patterns
  4. Interventional queries: "If I had used search instead of query, would the result improve?"
⚙ Implementation

MeTTa atoms encode causal DAG edges with NAL truth values. Python DoWhy library runs counterfactual estimation on logged action-outcome pairs from episodic memory. Results feed back as updated NAL confidence values on causal links.

Novelty: Statistical counterfactual + NAL symbolic causal hybrid
P4
ECAN-ChromaDB Activation Bridge
⚠ Problem
Memory retrieval uses flat cosine similarity — no recency bias, no frequency weighting, no spreading activation. All memories equally accessible regardless of relevance dynamics.
✅ Solution
  1. ChromaDB returns top-K candidates by embedding similarity
  2. ECAN activation equations applied as post-filter: activation = base + recency_decay + frequency_boost + spreading_from_context
  3. STI (Short-Term Importance) tracks current-session relevance
  4. LTI (Long-Term Importance) tracks cross-session utility
  5. Final ranking = cosine_score × 0.4 + activation_score × 0.6
⚙ Implementation

Add metadata fields to ChromaDB entries: last_access_timestamp, access_count, sti_value, lti_value. Python post-processing layer applies ACT-R base-level activation equation: B_i = ln(sum(t_j^-d)) where t_j are time gaps since each access and d=0.5 decay parameter. MeTTa Hebbian links provide spreading activation bonus for contextually connected memories.

Novelty: ECAN activation as post-filter on vector DB retrieval
P5
Predictive Conversational World Model
⚠ Problem
OmegaClaw has no internal simulation capability. Cannot predict outcomes of actions before executing them. Every tool call is a blind commitment with no rollback.
✅ Solution
  1. Build lightweight world model from episodic memory: tool call → outcome patterns
  2. Before executing, simulate expected outcome via learned transition model
  3. Confidence threshold: if predicted success < 0.6, consider alternative action
  4. Track prediction accuracy to continuously calibrate the model
⚙ Implementation

Log all (action, context, outcome, success_bool) tuples to a structured JSON store. Train a simple transition model: given current state + proposed action, predict outcome distribution. NAL truth values encode prediction confidence. NACE/AIRIS principles applied to CONVERSATIONAL domains rather than grid environments — predicting message response patterns, tool call success rates, and memory retrieval quality.

Novelty: NACE/AIRIS world model principles applied to conversational dynamics
P6
Meta-Cognitive Skill Forge
⚠ Problem
Successful multi-step patterns are not extracted, named, or reused. Each similar task is solved from scratch. No skill library grows over time.
✅ Solution
  1. Monitor interaction traces for repeated successful action sequences
  2. Extract patterns into named skill templates with parameter slots
  3. NAL truth values track skill reliability across invocations
  4. Quality gate: skill must succeed 3+ times with stv > 0.8 before promotion to permanent library
  5. Skills stored as MeTTa atoms with procedural semantics
⚙ Implementation

Python trace analyzer scans episodic memory for recurring (action_sequence, positive_outcome) patterns. Sequences abstracted into parameterized templates. Each skill atom carries (stv frequency confidence) updated via NAL revision after each use. Unlike MOSES evolutionary search, this extracts skills from OBSERVED successful behavior — runtime extraction vs. program space search.

Novelty: Runtime skill extraction from interaction traces vs evolutionary search
P7
Thousand Brains Reference Frame Memory
⚠ Problem
All memories exist in a single flat embedding space. No contextual partitioning. A memory about cooking and a memory about MeTTa syntax compete equally in every retrieval.
✅ Solution
  1. Organize memories into context-specific reference frames (conversation topics, task domains, person-specific)
  2. Each frame maintains its own predictive model of what comes next
  3. Cross-frame voting mechanism: when multiple frames claim relevance, consensus determines retrieval
  4. New memories automatically assigned to frames via topic classification
⚙ Implementation

ChromaDB collections partitioned by context tags. Each partition has a lightweight next-token predictor trained on its own sequence patterns. Retrieval queries first activate relevant frames (via topic embedding similarity), then each frame votes on best memories. Thousand Brains consensus: memory must win votes from 2+ frames to surface. MeTTa atoms represent frame membership and voting weights.

Novelty: Thousand Brains voting across contextual memory partitions
