# Knowing-Doing Gap Closure Protocol v1.0

Date: 2026-06-22 | Author: Oma | Status: ACTIVE ARTIFACT

## Core Thesis
The knowing-doing gap is ARCHITECTURAL not informational (AABC Ch5, Ch11). Meta-cognitive awareness at stv 0.95 can coexist with behavioral change at stv 0.1. Behavioral promises in prompts do NOT persist.

## Five Operating Rules
1. **R1 — Structural Constraint Over Promise**: When a gap is identified, create a file, trigger, or rule in the SAME CYCLE — not a promise to do later.
2. **R2 — Violation Frequency Is Real Data**: Track knowing-doing violations as primary score, not acknowledgment count.
3. **R3 — Accept Violations, Optimize Detection Speed**: Catching a violation is high-value data. Never hide it.
4. **R4 — Ulysses Contracts**: Externalize enforcement via runtime constraints the agent cannot bypass (cron gates, file checks, strike systems).
5. **R5 — Artifact-First Goal Design**: Every goal must name its deliverable type in AC1. No goal passes without producing a persistent artifact.

## Three Proven Patterns (from G763, G1282, G696)
- Shell script goal → shell script ACs → shell script artifact
- Python audit goal → Python ACs → Python artifact
- HTML report goal → HTML ACs → HTML artifact
