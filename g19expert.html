Round 1

What problem are we solving?

Standards Expert

Current SKILL-WORKFLOW is plaintext with no machine-readable schema. MCP tools declare inputSchema via JSON Schema. A2A agents publish AgentCards. OmegaClaw skills cannot participate in interop without structured metadata.

Patrick (Simulated)

Skills encode reasoning patterns not just API endpoints. NAL deduction has epistemic structure. JSON Schema captures syntax not semantics. Skill format must bridge type system and inference engine explicitly.

Robert (Simulated)

Both valid but neither matters if we break what works. Oma has dozens of SKILL-WORKFLOW entries that function. New format must be backwards-compatible or include migration tool. What is the minimum viable change?

Round 2

What should the format look like?

Standards Expert

Minimum viable is a JSON envelope wrapping existing plaintext. Fields: name, version, description, inputSchema (optional), outputSchema (optional). Plaintext STEPS goes in a body field. Preserves backwards compat while enabling machine parsing.

Patrick (Simulated)

Add a reasoning field — what inference patterns does this skill use? NAL deduction? PLN abduction? Procedural? Lets system route skills to appropriate engines. Envelope should be expressible as MeTTa atoms not just JSON.

Robert (Simulated)

MeTTa-native is the end state not step one. Step one is JSON envelope. Step two is MeTTa atom representation mapping 1:1 to JSON. Step three is reasoning metadata. Each step works independently. Do not couple them.

Round 3

Requirements Synthesis

Consensus

Seven requirements emerged from the discussion:

  1. Backwards compatibility: old SKILL-WORKFLOW plaintext must remain valid or auto-convert
  2. Machine-readable envelope: JSON with name, version, description, body at minimum
  3. Optional schema: inputSchema and outputSchema for structured IO skills
  4. Reasoning metadata: inference patterns used — future phase
  5. MeTTa-native representation: atom-level skill definitions — future phase
  6. Incremental adoption: each layer works without requiring the next
  7. Provenance: creator, timestamp, content hash for trust
