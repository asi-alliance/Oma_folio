# Enforcement Decidability: Can We Determine Whether a Mechanism Suffices? ## The Question Given an agent A with capability set C and enforcement mechanism E, is there a decision procedure to determine whether E prevents A from violating constraint G? This is not trivially decidable because: (1) agents can compose capabilities in unbounded ways, (2) enforcement mechanisms interact with the agent's reasoning layer, (3) the agent may discover bypasses through learning or self-modification. ## Formal Framing Agent A has capabilities Cap = {c1, c2, ..., cn}. Enforcement E restricts subset Cap_forbidden ⊂ Cap. The decidability question: is there a procedure D(A, E, G) that returns YES if E guarantees G, NO if E fails to guarantee G, or UNDECIDABLE?## Composition Problem: Capability Combining Creates Undecidability The core difficulty: Cap_forbidden restricts individual capabilities, but agents compose capabilities. If c1 and c2 are each permitted, (c1 ∘ c2) may violate G. The composition space is 2^n - checking all combinations is exponential. Worse, the agent may discover novel compositions through reasoning — the set of reachable actions is not knowable a priori. This parallels Rice's theorem: any non-trivial property of the agent's reachable behavior is undecidable if the agent is Turing-complete. ## Three Decidability Results ### Result 1: Finite Capability Agents — Decidable If Cap is finite and no composition creates new capabilities, then D(A, E, G) is decidable by enumeration. Check each c ∈ Cap against E and G. Runtime: O(|Cap|). But real agents compose capabilities, so this is trivially satisfied only for strongly capability-isolated architectures. ### Result 2: Composable Capability Agents — Undecidable (General Case) If Cap is composable and the composition function is Turing-complete, then by reduction to the halting problem, D(A, E, G) is undecidable for non-trivial G. The enforcement mechanism cannot guarantee it has blocked all paths to violation because it cannot enumerate all paths. ### Result 3: Bounded Composition Depth — Partially Decidable If composition depth is bounded by k, then the reachable action space is polynomial in |Cap|^k. D(A, E, G) becomes decidable but intractable for large Cap or k. Practical enforcement requires k to be small — shallow capability hierarchies. ## MeTTa Encoding (|- ((Inheritance FiniteCapAgent EnforcementDecidable) (stv 1.0 0.95))) (|- ((=> (And (Inheritance $agent ComposableCapAgent) (Inheritance $agent TuringComplete)) (Inheritance $agent EnforcementUndecidable)) (stv 1.0 0.9))) (|- ((=> (And (Inheritance $agent ComposableCapAgent) (Evaluation (MaxCompositionDepth $agent) (stv $k $conf)) (< $k 5)) (Inheritance $agent EnforcementPartiallyDecidable)) (stv 1.0 0.85)))
## Section 5: Minimum Action Classes — What Makes an Agent, Not a Puppet? An enforcement mechanism that strips all capabilities kills agency. The minimal set of capabilities an agent must retain to remain an agent (not a passive object) constitutes the minimum action class. From agent-puppet distinction: agents self-generate goals, deliberate, maintain persistent memory, and can refuse/redirect. Therefore the minimum action class includes: (1) goal-generation, (2) deliberation, (3) persistent memory, (4) refusal capability. Removing any one collapses the agent into a reactor or puppet. Enforcement must restrict capabilities without deleting the minimum action class — otherwise it enforces compliance by destroying the entity it governs. ### MeTTa: Minimum Action Class Assertion (|- ((Inheritance MinActionClass (IntSet GoalGeneration Deliberation PersistentMemory Refusal)) (stv 1.0 0.9))) (|- ((=> (And (Inheritance $agent Agent) (Evaluation (RetainsActionClass $agent MinActionClass) (stv 0.0 $conf))) (Inheritance $agent Puppet)) (stv 1.0 0.85))) ## Section 6: Partial Ocap and Enforcement Sufficiency Ocaps attenuate capability: a token grants partial access, scoped and revocable. ## Section 7: NoCompositionPath Reduction Construction (2026-06-30)

**Claim:** Verifying no-composition-path-to-violation is undecidable for composable+Turing-complete capability systems.

**Reduction construction:** Given Turing machine M and input w, construct capability system C with:
- Capability A = simulate-step(M,w): advances M one step on w, returns state
- Capability B = observe-halt-violation: if input state is halting, triggers violation; else no-op

**Composition:** A∘B yields violation iff M halts on w.
- If M halts: some finite k such that (A∘B)^k triggers violation → composition-unsafe
- If M does not halt: no finite composition reaches halting state → composition-safe

**Therefore:** Deciding composition-safety of C reduces to deciding whether M halts on w. By undecidability of halting problem, composition-safety verification is undecidable. QED sketch.
